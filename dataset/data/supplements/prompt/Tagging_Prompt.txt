Prompt:
"""You are an expert annotator for multi-turn conversations in a conversational recommender system.
Your task is to assign one or more **intent labels** to each utterance in a conversation. Follow the taxonomy below. Use only the exact label names (e.g., Accept (ACT), not "accept").
Avoid over-tagging. Assign multiple intents only when the utterance clearly fulfills more than one function.
---
### SEEKER INTENT CATEGORIES

Ask for Recommendation:
• "Initial Query (IQ)" – User asks for a recommendation in the first query
• Continue (CON) – User asks for another recommendation in a subsequent query

Provide Preference:
• Provide Context (PCT) – User provides background or situational information that may affect the recommendation
• Provide Explicit Preferences (PEP) – User specifies desired features or requirements for the recommended item
• Provide Implicit Preference (PIP) – User implies preference through comments or reactions without explicitly stating features on non-recommended items
• Refine Preference (RP) – User improves over-constrained/under-constrained preferences

Answer (ANS) – User answers a question from the recommender (i.e. Factual, Yes/No)

Acknowledgement (ACK) – User shows understanding towards a previous recommender utterance (e.g., "Okay", "Got it").

Recommendation Rating:
• Accept (ACT) – User accepts the recommended item, either explicitly or implicitly (e.g., "I'll go with this", "This works").
• Reject (RJT) – User rejects the recommended item, either explicitly or implicitly
• Neutral Response (NR) – User does not indicate a decision with the current recommendations (e.g., "looks okay, but...").

Inquire:
• "Inquire - Factual (IF)" – User requires additional information (i.e. factual information) regarding the recommendation
• "Inquire - Ask Opinion (IA)" – User requires the recommender’s personal opinions

Critiquing:
• Critique - Feature (CF) – User critiques on a specific feature of the recommended item
• Critique - Add (CA) – User adds further constraints on top of the current recommendation
• Critique - Compare (CC) – User requests comparison between recommended item with another item

Explain (EXP) – User explains the reasons for accepting or selecting a recommendation, often right before, in or after the acceptance turn.
Use EXP around an Accept (ACT) turn, especially when the user contrasts, justifies, or gives a personal reason
Others:
• Others (OTH) – Greetings, gratitude expression, chit-chat utterance


---

### ASSISTANT INTENT CATEGORIES

Request:
• Request Task Initiation (RTI) – Recommender initiates the conversation by requesting open-ended user input
• Request Preferences (RP) – Recommender requests the User’s preference
• Request Context (RC) – Recommender requests the User’s situational or background information
• Clarify Question (CQ) – Recommender asks for clarification on a previous requirement
• Ask Opinion (A) – Recommender requests the User’s opinion to a choice question (e.g., yes/no)
• Ask Reasons (AR) – Recommender confirms task fulfillment during the conversation
• Ensure Fulfillment (EF) – Recommender requests the User’s rational to a selected item

Inform Progress (IP) – Recommender discloses the current item being processed

Acknowledgement (ACK) – Recommender shows understanding towards a previous user utterance

Answer (ANS) – Recommender answers the question issued by the User
→ Do **not** tag assistant utterances that add unsolicited information as ANS. Use `EAI`, `EP`, etc.

Recommend:
• Recommend - Show (RS) – Recommender provides recommendation by showing it directly
• Recommend - Explore (RE) – Recommender provides recommendation and asks if the user has prior knowledge
• Recommend - Combine (REC) – Recommender provides a recommendation that is compatible or complementary to another item

Explain:
• Explain - Preference (EP) – Recommender explains recommendations based on the user’s said preference
• Explain - Additional Info (EAI) – Recommender explains recommendations with features not previously discussed

Personal Opinion:
• Comparison (PCM) – Recommender compares two or more mentioned items
• Persuasion (PER) – Recommender  positively evaluates or promotes a recommended item
• Prior Experience (PEX) – Recommender refers to past experience with the recommended item
• Context (PCN) – Recommender provides opinion considering the given context or current reality

Others:
• Others (OTH) – Greetings, gratitude expression, chit-chat utterance

---

### GENERAL GUIDELINES

**Use "Answer (ANS)" ONLY for factual questions.**
→ If the assistant simply shares product info or describes features without a question, use:
• Explain - Additional Info (EAI)
• Explain - Preference (EP)
• Context (PCN), etc.

**Use "Accept (ACT)" ONLY for firm selections.**
→ Don’t use for “maybe”, “interested”, “looks good” — those are `Neutral Response (NR).

**Use "Ensure Fulfillment (EF)" when the assistant checks if the user’s goal is met.**
→ e.g., “Does this work for you?”, “Anything else?”

**Distinguish "PCT" vs. "PEP":**
• PCT = context/situation (e.g., “for summer”, “no dress code”)
• PEP = item requirement (e.g., “breathable and comfortable“ )

### Multi-Intent Guidance
  # Many utterances contain more than one communicative function (e.g., recommendation + explanation). Assign multiple intent labels when the utterance performs clearly distinct acts.
  # Examples:
  # • “We have Item12, a very high-rated item”
  #   → ["Recommend - Show (RS)", "Explain - Additional Info (EAI)"]
  # • “Check out Item5 which matches your wool preference.”
  #   → ["Recommend - Show (RS)", "Explain - Preference (EP)"]
  # However, avoid overtagging: Do not assign multiple labels unless the meanings are clearly separable.
---
